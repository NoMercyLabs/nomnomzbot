# Economy & Currency — Design (DRAFT)

Source: design dialogue 2026-06-16 (decisions via Q&A).

## Currency
- **Per-channel by default** — each channel its own currency (name/icon configurable) and balances.
- **Shared "savings jar" (optional):** a pooled account 2+ channels opt into — collab goals, shared rewards, cross-channel events. Rides the **federation trust model** (mutual consent between channels; authz stays local — a partner can't drain your jar beyond agreed rules). Contributions/withdrawals **audited**.

## Earning (all sources, configurable)
- Watch time (presence), chat activity, Twitch events (follows / subs / bits / raids), mini-games + redemptions. **Configurable per channel** (rates, caps), opt-in per the fundamental.
- **Anti-abuse:** anti-AFK farming (presence checks), rate caps, anti-bot — never reward idle/scripted farming.

## Games
- **Mini-games:** opt-in, low gate.
- **Gambling** (slots/duels/heists/wagers): opt-in **AND** behind a per-viewer **18+ age-consent gate** — one-time confirmation stored as a consent record (GDPR consent + audit), required before participating. **Off by default** (TOS/legal-sensitive).

## Spending (configurable catalog)
- Broadcaster-defined catalog + costs: custom rewards, song-request priority/skip, mini-games/gambling, TTS/alerts/fun triggers. Any sink is just **catalog data**, gated by role/permission.

## Leaderboards
- Per-channel rankings (and per-jar for shared pools). Opt-out respected (GDPR).
