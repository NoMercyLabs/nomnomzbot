# Widgets & Overlay Editor — Design (DRAFT)

Source: design dialogue 2026-06-16 (decisions via Q&A). The widget system, in-app editor, OBS pipeline, and sharing model.

## Authoring
- **Multi-framework:** users build widgets in **Vue / React / Svelte / vanilla** — their pick, via a shared build pipeline (esbuild).
- **In-app editor:** VSCode-web (Monaco), compile-on-save — the same editor stack as the custom-command code tier (one editor investment).

## Build + OBS pipeline
- **Server-side build:** on save the bot compiles the widget (esbuild) → on success **pushes a reload** to the OBS browser source via the **OverlayHub** (SignalR). Compile-on-save → live reload, no manual refresh.
- Build errors surface in the editor (validate-on-save), never silently.

## Live data
- Widgets receive events over the **OverlayHub** (SignalR) — the bot pushes **allowlisted** events (per the event-allowlist + message-parsing model). Widgets subscribe to the types they need.

## Serving
- **Self-host:** served from localhost (OBS on the same box) or the operator's tunnel for remote.
- **SaaS:** served from our edge; the browser-source URL carries the per-channel `OverlayToken`.

## Sharing — curated, verified, GitHub-sourced
- **First-party templates:** alerts, now-playing, chat, goal bars, etc. — fully trusted, one-click.
- **Verified community gallery:** users submit via a **GitHub link**; we **review + verify** before listing. GitHub source gives provenance + auditability (open, inspectable) + **version pinning** (pin to a commit/tag; re-verify on update — never auto-pull HEAD).
- **Trust tiers:** first-party (trusted) > verified-community (reviewed, pinned) > arbitrary/unverified (self-host-only, at the operator's own risk; not in the SaaS gallery).

## Safety (defense-in-depth)
- A widget runs in the streamer's own OBS, but even verified/shared widgets render under a **sane CSP + sandbox** — verification reduces risk, sandboxing contains the rest.
- Any user/remote content a widget renders flows through the same **XSS-zero token pipeline** as chat messages.
